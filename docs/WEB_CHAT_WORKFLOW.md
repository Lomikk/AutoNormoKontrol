# Работа через браузерный ИИ-чат и Aider

Этот сценарий предназначен для пользователя, у которого нет агента с прямым
доступом к файлам. Движок остаётся локальным, а браузерный ИИ только предлагает
изменения. Все файлы документа находятся внутри `Workspaces/<работа>/`.

## Что создать заранее

Новую работу создают из корня движка:

```powershell
.\AutoNormoKontrol.cmd new "Название работы"
```

После этого все операции с документом выполняются в созданном workspace:

```powershell
cd ".\Workspaces\Название работы"
.\AutoNormoKontrol.cmd draft
```

Корень движка не содержит рабочее содержание и не поддерживает сборку документа.

## Что передать браузерному ИИ

Минимальный набор зависит от задачи. Не отправляйте весь движок.

Всегда полезны:

- локальные `AGENTS.md` и `GEMINI.md`;
- `project.yaml`;
- точная формулировка задачи;
- полный текст последней компактной ошибки сборки.

Для правки содержания добавьте:

- нужный файл из `content/`;
- соседний раздел, только если он нужен для связности;
- `metadata.yaml` или `bibliography.bib`, если задача касается данных или ссылок;
- относящиеся к задаче материалы из `research/` и `assets/`.

Не отправляйте без необходимости:

- `EngineRoot`, `profiles/`, `schemas/`, `scripts/`, `sources/` и `tests/`;
- сгенерированный `build/`;
- весь нормативный PDF;
- журналы `compliance/semantic-review.yaml` и
  `compliance/external-acceptance.yaml` во время обычного написания.

Если браузерный ИИ просит внутренний файл движка, сначала уточните, зачем он
нужен. Обычная правка документа не требует доступа к реализации профиля.

## Цикл copy-paste

1. Передайте ИИ минимальный набор файлов и попросите вернуть unified diff либо
   полные версии только изменяемых файлов.
2. Проверьте, что предложены изменения только внутри workspace.
3. Примените изменения локально.
4. Запустите:

   ```powershell
   .\AutoNormoKontrol.cmd draft
   ```

5. При ошибке отправьте ИИ сообщение целиком, включая код, `file:line`, ID
   объекта и предложенный исходный файл, если они показаны.
6. Повторяйте цикл до успешного Draft и визуальной проверки PDF.

Проверка самого движка командой `check` выполняется разработчиком из корня
движка. Пользователю, который пишет работу, она обычно не нужна.

## Как формулировать запрос

```text
Работай только с переданными файлами workspace.
Не выдумывай источники, персональные данные и внешние решения.
Не добавляй raw TeX/HTML и ручную вёрстку в content/*.md.
Верни unified diff.
После применения diff я пришлю результат AutoNormoKontrol.cmd draft.
```

Для библиографической ошибки передайте главу с цитированием,
`bibliography.bib` и точное сообщение Draft. Для рисунка или таблицы передайте
соответствующую главу, manifest ассетов и затронутые исходники ассета.

## Aider как необязательный локальный редактор

Aider можно запускать в каталоге конкретной работы, а не в корне движка. Он не
используется для разработки AutoNormoKontrol и не получает доступ к
`EngineRoot`.

Для документного workspace карта репозитория обычно не нужна:

```powershell
aider --map-tokens 0
```

Пользователь вручную задаёт минимальный набор файлов:

```text
/read-only AGENTS.md
/read-only GEMINI.md
/read-only project.yaml
/add content/00-introduction.md
```

При переходе к другой задаче ненужные файлы следует убрать через `/drop`, затем
добавить новый целевой файл. Команды `/add`, `/read-only` и `/drop` являются
возможностями Aider, а не публичными командами AutoNormoKontrol.

После законченной правки Aider должен один раз выполнить локальную команду
`AutoNormoKontrol.cmd draft` или пользователь запускает её сам. Автоматического
управления процессом и контекстом Aider в MVP нет.

## Финальная проверка

Успешный Draft означает, что промежуточный PDF собран и программные проверки
пройдены. Он не равен Strict.

Перед Strict нужны:

1. стабильный успешный Draft и его `content-hash`;
2. полный semantic review именно этого снимка;
3. реальные внешние подтверждения в external acceptance;
4. локальный запуск `AutoNormoKontrol.cmd strict` с exit code 0.

Пользователь вправе экспортировать Draft с явным предупреждением, но программа
не должна представлять такой результат как прошедший Strict.

## Конфиденциальность

Не отправляйте в браузерный сервис персональные данные, закрытые методички или
неопубликованные материалы без осознанного разрешения. При необходимости
используйте обезличенные значения, а реальные реквизиты внесите локально.

Структурированная краткая диагностика с устойчивыми кодами и координатами
исходника запланирована как R1.4b. До её завершения следует передавать ИИ полный
фрагмент ошибки, не угадывая файл по номеру страницы PDF.
