# Контракт агента рабочей области

Эта папка содержит одну курсовую работу. Перед изменением содержания полностью
прочитай `guide/profile-system-prompt.md`: это локальный точный снимок системной
инструкции выбранного профиля. Если файл отсутствует или движок сообщает о
несовпадении его хэша, не продолжай работу и попроси восстановить workspace.

Центральный AutoNormoKontrol хранит
профиль, нормативный реестр, фильтры и LaTeX-реализацию отдельно. Не выходи из
рабочей области и не пытайся изменять движок при обычном написании документа.

## Что можно изменять

- `content/*.md` — содержание по устойчивым крупным разделам;
- `metadata.yaml` — только подтвержденные реквизиты пользователя;
- `bibliography.bib` — только реально использованные и проверенные источники;
- `assets/**` — исходные данные, изображения и объявленные в manifest ассеты;
- `research/**` — исследовательские заметки, не являющиеся текстом работы.

Не изменяй без отдельного запроса пользователя:

- `format-spec.yaml` и локальный `AutoNormoKontrol.cmd`;
- `compliance/semantic-review.yaml` и `compliance/external-acceptance.yaml`;
- производные каталоги `build/**` и `output/**`.

`project.yaml` разрешено менять только при явно поставленной задаче добавить,
удалить или переставить крупную главу. В этом случае синхронно обнови точный
упорядоченный массив `document.content`; не добавляй туда недостающие файлы и не
выводи порядок сканированием каталога.

## Обязательные ограничения

1. Не выдумывай тему, дисциплину, персональные данные, задание, подписи, даты,
   источники, DOI, страницы, результаты, расчеты, апробацию и эффективность.
2. `content/*.md` содержит только семантический Pandoc Markdown. Запрещены raw
   TeX/HTML, ручные номера и визуальная подгонка (`vspace`, `hspace`,
   `fontsize`, `newpage`, `pagebreak`, изменение полей и абсолютные координаты).
3. Заголовки не содержат ручных номеров. Каждый нумерованный раздел завершается
   ненумерованным подразделом `Выводы по разделу <номер словом>`.
4. На рисунок, таблицу, формулу и приложение сначала ссылаются в тексте, а
   затем размещают объект. Используй только нотацию из
   `guide/notation-examples.md` и стабильные ID `fig:`, `tbl:`, `eq:`, `app:`.
5. Не набирай библиографический список вручную. Используй `[@ключ]`, реальную
   запись в `bibliography.bib` и существующий блок `::: bibliography`.
6. Не присваивай смысловому аудиту `pass` без полного аудита текущего снимка и
   штатного `content-hash`. Не присваивай внешней приемке `accepted`: такое
   решение принимает только уполномоченный человек на основании реального
   свидетельства.

## Рабочий цикл

После законченной правки запусти из корня этой работы:

```text
AutoNormoKontrol.cmd draft
```

Успешный Draft означает, что программные проверки и PDF-postflight прошли. Он
не подтверждает смысловую корректность и не заменяет внешнюю приемку. `strict`
запускается только для согласованной контрольной версии; пользователь вправе
работать и сдавать Draft, понимая его ограничения.

Исправляй исходный файл, указанный диагностикой. Не редактируй промежуточный
`build/coursework.tex` или PDF. Если корректное требование нельзя выразить
разрешенной нотацией, остановись и объясни, что требуется изменение профиля
разработчиком, а не обход в содержании.
