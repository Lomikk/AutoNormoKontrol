# Разрешенная нотация содержания

Этот файл является справкой и не включается в PDF. Копируйте только нужный
пример в подходящий `content/*.md`, заменяя ID, подписи, пути и данные.

## Библиографическая ссылка

Сначала добавьте проверенную запись в `bibliography.bib`, затем используйте ее
ключ:

```markdown
Требование приведено в стандарте [@source-id].
```

## Рисунок

Первое упоминание непосредственно предшествует рисунку. Для графика обязательны
правдивые атрибуты осей; готовый PDF создается asset pipeline по
`assets/manifest.json`.

```markdown
Результат показан на рисунке [@fig:result].

![Содержательное наименование](build/assets/result.pdf){#fig:result kind=graph x-axis="Обозначение оси X" y-axis="Обозначение оси Y, единица" width=80%}
```

Повторная ссылка имеет вид: `см. рисунок [@fig:result]`.

## Таблица

```markdown
Параметры приведены в таблице [@tbl:parameters].

| Параметр | Значение |
|:--|:--|
| [наименование] | [подтвержденное значение] |

: Параметры исследования {#tbl:parameters}
```

Повторная ссылка имеет вид: `см. таблицу [@tbl:parameters]`.

## Формула с новыми обозначениями

```markdown
Показатель определяется по формуле [@eq:quality].

::: {#eq:quality .equation symbols="Q,N_ok,N_all"}
$$
Q = \frac{N_{\mathrm{ok}}}{N_{\mathrm{all}}},
$$
:::

::: {.equation-where for="eq:quality"}
- $Q$ — значение показателя
- $N_{\mathrm{ok}}$ — количество элементов, удовлетворяющих условию
- $N_{\mathrm{all}}$ — общее количество элементов
:::
```

Символы в `symbols` и пояснениях идут в порядке первого появления в формуле.

## Приложение

Ссылка в основной части должна стоять раньше блока приложения:

```markdown
Исходные данные приведены в приложении [@app:source-data].
```

Сам блок помещают в `content/99-appendix.md`:

```markdown
::: {#app:source-data .susu-appendix letter="А" title="Исходные данные" kind="информационное"}

[Поместите только связанный с работой вспомогательный материал.]

:::
```

Буквы приложений следуют порядку первых ссылок. Любая таблица, формула или
рисунок внутри приложения также получает собственный ID и ссылку.
