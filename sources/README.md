# Нормативный корпус источников

`sources/` — библиотека первоисточников и их воспроизводимых производных
данных. Это не каталог содержимого конкретной работы и не каталог профилей.

Структура для импортируемых документов ЮУрГУ:

```text
sources/
  susu/
    provenance/       исходный список файлов со страницы ЮУрГУ
    catalog.json      машинный каталог всего корпуса
    sto-XX-YYYY/
      original/       неизменённый нормативный PDF
      attachments/    исходные DOC и проверенные DOCX/PDF-конвертации форм
      derived/        текстовые копии и технические сведения Poppler
      manifest.json   хэши и связь оригиналов с производными файлами
```

`Документы ЮУрГУ/` используется только как временная локальная входная папка
скачивания и не версионируется. При новом импорте её создают рядом с
репозиторием либо повторно в его корне, после чего запускают детерминированный
импорт:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/import-source-corpus.ps1 -DryRun
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/import-source-corpus.ps1
```

Импортёр не интерпретирует требования, не создаёт профиль документа и не
изменяет действующие `requirements.json`, `metadata.yaml` или `content/`.
При существующем файле он сверяет SHA-256 и отказывается перезаписывать другой
источник.

Активный профиль СТО ЮУрГУ 21–2008 использует канонический оригинал и текст из
`susu/sto-21-2008/`. Временные корневой PDF, отдельный TXT и старый source
manifest после проверки эквивалентности удалены; fallback на них отсутствует.
