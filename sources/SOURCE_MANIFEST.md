# Источник требований

## СТО ЮУрГУ 21–2008

- Исходник: `../Normokontrol-2008-2017.pdf`
- SHA-256: `8B5A86D952EDB617DEA530F97C814182D6EA01417D8E05153086CE5DC3306153`
- Объём: 56 страниц A4; PDF 1.6; документ имеет собственный текстовый слой.
- Текстовая копия: `sto-susu-21-2008.txt`
- SHA-256 текстовой копии: `F18DD346D94D2A9B4C79BAEA7CEAB40E741E4736374BDD1F7C323C906BEDC910`
- Воспроизводимая команда извлечения:

  ```powershell
  pdftotext -layout -enc UTF-8 Normokontrol-2008-2017.pdf sources/sto-susu-21-2008.txt
  ```

- Проверенный инструмент: Poppler `pdftotext` 25.02.0.

OCR и LLM-реконструкция для этого файла не применялись: они внесли бы новый
слой вероятностных ошибок без необходимости. PDF и извлечённый текст сохраняются
вместе; ссылки на пункты сверяются с PDF, а текстовая копия используется для
поиска и машинной инвентаризации.

`requirements.json` является реестром интерпретаций требований, но не заменяет
первоисточник. При расхождении приоритет имеет указанная выше версия PDF.
